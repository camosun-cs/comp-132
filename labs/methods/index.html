Task 1: Change Machine

Suppose that you want to develop a program that classifies a given amount of money into smaller monetary units. The program lets the user enter an amount as a double value representing a total dollar and cents, and outputs a report listing the monetary equivalent in dollars, quarters, dimes, nickels and pennies as shown below:

This program should be able to keep asking for end user for a new number to convert until a negative number is entered

Approach
  • Prompt the user to enter the amount such as 11.56
  • Convert amount into cents (1156)
  • Divide by 100 to find dollar amount : 11
  • Divide the remainder by 25 to find the number of quarter: 2
  • Devide the remainder by 10 : 0 dime
  • Devide remainder by 5: 1 nickel
  • The remaining are pennies: 1

Task 2: Rank Teams

Using Switch statement, write a program that rank teams according to the number assigned to the classification below. The program should keep prompting the user to enter the team number, before performing the ranking , until the user enters a negative number

  • 7-10: Group A
  • 4-6: Group B
  • 2-3: Group C
  • 0-1: Group D
  • Any other positive number: there no group corresponding to this number.

Task 3: Generate Fibonacci Numbers

Write a program that output the Fibonacci numbers that are less than the number specified by the end user. The program should keep asking for a new upper limit, until the user enters a negative number or a zero. Each Fibonacci number is the sum of its two predecessors. The first two Fibonacci number are 1 and 1. Thus the sequence begins with: 1,1,2,3,5,8,13,21,34,...

Task 4: Lottery Game II

Expanding on the lottery program that was designed in the previous class, make some modification so that the computer can help the user guess the right number. The computer will generate a random number and give the user an opportunity to guess the number

  • If the number is too low, the computer will let the user know that the number is too low and prompt the user to guess another number
  • If the number is too high, the computer will let the user know that the number is too low and prompt the user to guess another number
  • When the user guess the right number, the program will congratulate the user and let the user know how many trials it took for them to guess the right number, then let them know how much they won, before terminating
  • If the user guess the right number on the first trial, the price is $10,000
  • For any subsequent trial, the price reduces by $1,000 per trial
  • Once the price reaches $1,000, any subsequent trial will be worth $10, when the user guess the right number
