Task

Continuing to work on your GUI from the last exercise, use ActionListener's to make your buttons do something useful.

How to implement and use an ActionListener

You can augment one of your existing classes (or create a brand new class) to use as an ActionListener by following these steps:

  1. Add imports for java.awt.event.ActionListener and java.awt.event.ActionEvent at the top of your file.
  2. Add implements ActionListener to your class definition.
  3. Add a definition for the ActionListener#actionPerformed(ActionEvent) method.

For example:


				import java.awt.event.ActionListener;
				import java.awt.event.ActionEvent;
				class MyListeningClass implements ActionListener {
					public void actionPerformed(ActionEvent e) {
						// make something happen here!
					}
					...
				}

Then, provide an instance of your class as the argument to the addActionListener method of one or more of your JButton's: 


				ActionListener myListenerInstance = new MyListenerClass();
				myButton.addActionListener(myListenerInstance);

You can use very similar steps to respond to many other events in your GUI by creating and adding different kinds of EventListeners. For example, extend TextListener and use the addTextListener method to react to content changes from JTextField and JTextArea elements, or ChangeListener and the JSpinner#addChangeListener method to notice changes from a JSpinner.

Here are some ideas for things you can make your UI do:

  • Print a triangular matrix of *'s (using code from Lab 4) inside a JTextArea when the "Draw Matrix" button in pushed.
  • Display the result of adding or subtracting the values from two JSpinner's in the Calculator section when the "+" or "-" buttons are pushed.
  • Make the "Add" button in the Personal Information section create a Person object (from Lab 7) using the provided information, and display it in a list along with any others you've made so far.
  • Remove a previously created Person object from the list in the Person Information section when the "Remove" button is pushed.
  • Clear the answer output in the Calculator section any time the value of either of the JSpinner's is changed.
  • Change the border color of the Display Results section every time the content of the JTextArea changes.

Experiment with different ways of creating ActionListeners. Try using your existing GUI-building class as an ActionListener to start, then try creating new classes that are responsible for different actions. You may find it helpful to keep the GUI construction code separate from the ActionListener in some cases, or better to keep them together in others!
