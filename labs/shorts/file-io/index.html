Task

Write a program that reads data from a file, computes something interesting, and writes the results to another file.

There are many ways to do this, but these classes from the Java standard library will be particularly helpful:

  • javax.swing.JFileChooser
  • java.io.File
  • java.util.Scanner
  • java.io.PrintStream
  • java.io.FileOutputStream

You can use Scanner to read data from files in the same way you've used it to read user input from the keyboard. The only difference will be the source of the Scanner's data, given in its constructor.

You should also already be familiar with how to use PrintStream: we've been using one since the beginning of this course called System.out. For this task, you'll need to instantiate one yourself, giving it the file to output to as a parameter to the constructor.

Working with file paths in NetBeans can sometimes be a little tricky. JFileChooser provides a graphical file picker that will make it easier to get started. If you use any literal Strings for paths on Windows, make sure that the \ back-slash directory separators are escaped (eg. "C:\\file\\path"), since this character has special meaning inside of Java String values.

There are many things out of your control that could go wrong while working with files: OS permissions can prevent you from reading or writing data, disks can run out of space, users can unexpectedly delete thnigs, etc... Java signals these potential problems using Exceptions. We haven't had time to talk about Exceptions in this course, so for now you can avoid the issue by telling Java that you don't mind letting your program crash if one of these problems occurs. To do this, add throws Exception to any method that works with Files like this:


			public static void main(String[] args) throws Exception {
				// do work with Files, and just crash if anything goes wrong!
			}
