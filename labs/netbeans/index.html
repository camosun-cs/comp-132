Learning Outcomes and Introduction

Setup: Installing NetBeans

If you are working in a computer lab at Camosun, simply log in using your Camosun ID. NetBeans will already be installed for you and you can skip this section.

If you would like to use NetBeans on your own computer, you'll need to install it yourself! Follow the instructions in this section to get it set up.

  1. Go to netbeans.apache.org in a web browser.
  2. Choose Download from the main navigation menu.
  3. Choose the link for "binaries." At the time these instructions were written, the link looks like "incubating-netbeans-10.0-bin.zip."
  4. Click the suggested download link at the top of the page, and wait for the download to complete.

  5. If you are using MacOS,

    1. Double-click the downloaded ZIP file to extract its contents.
    2. Open the "netbeans" folder, and open the "bin" folder inside of it.
    3. Right-click the "netbeans" file with the black icon and choose "Open".
    4. A security warning likely appear; this is normal! Choose "Open" to proceed.
    5. In the future, you can double-click this file to open NetBeans and the security warning will no longer appear.

  6. If you are using Windows,

    1. Open the downloaded file.
    2. Choose "Extract All" from the ribbon and follow the prompts.
    3. Open the extracted "netbeans" folder, and open the "bin" folder inside of it.
    4. Double click "netbeans64.exe" to open NetBeans for the first time.
    5. If this doesn't work, try "netbeans.exe" instead.
    6. In the future, NetBeans should appear in your start menu, but you can always start it by opening this file.
  7. NetBeans may ask you to download Java SE components the first time it is opened, or the first time you start a new project. Follow the prompts to complete the installation.

Task 1: NetBeans and Java basics

  1. Launch NetBeans
  2. In NetBeans, go to File > New Project
  3. In the New Project window, select Java from the Categories pane and Java Application from the Projects pane, then click Next
  4. Change the Project Name to java-lab-assignments, or something similar. This project will be used for all of your lab assignments for this course.
  5. Change the Project Location to your M drive. This will ensure that your files are available from any lab computer.
  6. Uncheck the Create Main Class checkbox.
  7. Click Finish.
  8. Outside of NetBeans, go back to your M drive. Notice that NetBeans has created a new folder here with the name of your project.

  9. Open the project folder, and navigate to the src folder inside. This is where the Java files you write will be stored (your "source code"). It's empty right now because we haven't yet written any code!

    What are all these other files?

    The build.xml tells NetBeans how to compile and package your project's code.

    The nbproject folder contains files that NetBeans uses to keep track of your project settings, as well as additional build scripts referenced by the build.xml file.

    The manifest.mf contains metadata about your project that Java may need when your code runs.

    Why use packages?

    Packages help to keep your code organized by keeping related files together. You will store all of your Java code in this course together in one project, but should use a separate package (really just a folder inside of src) for each lab assignment to keep their files separated and make it easy to ZIP them up for submission to D2L.

    As you've seen, NetBeans can manage your files from within the IDE, or you can manage them directly in the src folder. Either approach is fine!

  14. The little red exclamation badges in the Projects panel and editor pane are indicating that the NetBeansIntro.java file we added still has errors. We cannot compile or run our code until all of these warnings are fixed. Hover your mouse over the red icons to find out what the problems are, or open the Action Items panel (choose Window > Action Items from the menu) for a full list.

    What does this error mean!?

    The error messages you get from the Java can be quite cryptic, especially when you're just starting to program. They tend to be very short, and use a lot of specialized jargon.

    Do not depair! You will learn what a lot of this jargon means throughout this course, and as you get more experienced, just knowing where the error occurs can often be enough for you to spot the problem.

    In the meantime, the most common errors you will see when you are just beginning will be:

    … expected
    not a statement
    reached end of file while parsing
    Something is wrong with your syntax. Check for missing semi-colons and braces.
    cannot find symbol
    You probably misspelled something. This can also happen if you've forgotten to define a variable before using it.
    incompatible type
    You've probably forgotten to use quotes around a string value, put quotes around a number when you shouldn't have, or used single-quotes when you should have used double-quotes. This error will come up for many more situations as you learn to do more with Java.

    Mindprod.com has a nice big list of compilation errors and their likely causes that might also be helpful.

  15. Fix the error in the sample code by removing the line that says "Opps!" (line 4), and save your changes.
  16. Look back at your project folder on your M drive. There should now be a build folder next to your src in the project directory. This is where NetBeans stores your compiled code. By default, NetBeans will automatically compile your Java source code into runnable .class files any time you save the file or attempt to run it.

  17. Run the example program by opening it in the NetBeans editor and choosing Run > Run File from the menu.

    Why can't I use the big green Run Project button?

    Projects are normally meant to store just one program, and so this button will only run the main method in one of your Java files (called the "main class"). Right now that's fine, but as you add more files to this project it will quickly become less useful.

    If you really like using that button instead of the menu or hotkeys, you can change which class will be run at any time in the settings for your project. Right-click the project name in the Projects panel and choose Properties. Choose Run from the Categories pane and change the Main Class setting as appropriate.

  18. If everything is working normally, you should the following appear in the output panel:

    run:
Success?
BUILD SUCCESSFUL (total time: 0 seconds)

    The black text is the output from this program. The coloured bits are messages from NetBeans letting you know where this output came from, and whether everything was working normally or not.

Submission

Upload a ZIP file containing your full src directory to D2L.

To create a ZIP file on MacOS:

  1. Right-click (or control-click) a folder in the Finder
  2. Choose Compress "…"
  3. Press enter to start renaming the ZIP file
  4. Type a name for the ZIP file and press enter

To create a ZIP file on Windows:

  1. Right click a folder in the file explorer
  2. Choose Compressed (zipped) folder from the Send to submenu
  3. Type a name for the ZIP file and press enter

Most lab assignmets will require you to upload just one of your source code packages to D2L, and also hand in a printed copy for marking. This assignment will be given a grade for completion only, so a printed copy is not needed.

To print a source code file...

  1. Open the file in the NetBeans editor
  2. Choose File > Print… from the menu
  3. Adjust the zoom options if necessary, then click Print
