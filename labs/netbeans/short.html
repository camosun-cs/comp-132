Learning Outcomes and Introduction

In this lab assignment you will learn about:

  • Creating a project in NetBeans.
  • Adding and organizing source files.
  • Compiling and running a Java program.
  • Writing a simple program that prints some text on screen.
  • Creating a ZIP file to submit your work to D2L.

Setup: Installing NetBeans

If you are working in a computer lab at Camosun, simply log in using your Camosun ID. NetBeans will already be installed for you and you can skip this section.

If you are working on your own computer, you'll need to install a JDK and NetBeans yourself! Follow the instructions in this section to get it set up.

If you are using Windows:

  1. If you already have Java 14 or later installed, remove it or ask for assistance before continuing these instructions.
  2. Download the OpenJDK-11 installer for Windows, open it, and follow the prompts to complete the installation.

  3. Download the NetBeans 11.3 installer for Windows, open it, and follow the prompts to complete the installation.

    You will not need the "Jave EE" or "PHP" components for this course, so you may omit them from the NetBeans installation if you'd like to conserve some disk space.

If you are using MacOS:

  1. Install the latest version of a Java SE Development Kit (JDK). You can download an installer from Oracle, or any other trusted source, such as Homebrew.
  2. Download the NetBeans 11.3 installer DMG for MacOS, open it, run the installer package inside, and follow the prompts to complete the installation.

If you are using Linux, use your operating system's package manager to install the latest version of NetBeans IDE. An appropriate version of Java will automatically be installed for you.

Task 1: NetBeans and Java basics

  1. Launch NetBeans
  2. In NetBeans, go to File > New Project
  3. In the New Project window, select Java with Ant from the Categories pane and Java Application from the Projects pane, then click Next
  4. Change the Project Name to comp-132, or something similar. This project will be used for all of your lab assignments for this course.
  5. If you are working in a computer lab at Camosun, change the Project Location to your M drive. This will ensure that your files are available from any lab computer. Otherwise, choose a location that you can find easily later, such as in your home directory.
  6. Uncheck the Create Main Class checkbox.
  7. Click Finish.
  8. Outside of NetBeans, find your Project folder using the File Explorer (on Windows) or Finder (on Mac).

  9. Find the src folder inside of your Project folder. This is where the Java files you write will be stored (your "source code"). It's empty right now because we haven't yet written any code!

    What are all these other files?

    The build.xml tells NetBeans how to compile and package your project's code.

    The nbproject folder contains files that NetBeans uses to keep track of your project settings, as well as additional build scripts referenced by the build.xml file.

    The manifest.mf contains metadata about your project that Java may need when your code runs.

    Why use packages?

    Packages help to keep your code organized by keeping related files together. You will store all of your Java code in this course together in one project, but will use a separate package (really just a folder inside of src) for each lab assignment to keep their files separated and make it easy to ZIP them up for submission to D2L.

    As you've seen, NetBeans can manage your files from within the IDE, or you can manage them directly in the src folder. Either approach is fine!

  14. The little red exclamation badges in the Projects panel and editor pane are indicating that the NetBeansIntro.java file we added still has errors. We cannot compile or run our code until all of these warnings are fixed. Hover your mouse over the red icons to find out what the problems are, or open the Action Items panel (choose Window > Action Items from the menu) for a full list.

    What does this error mean!?

    The error messages you get from Java can be quite cryptic, especially when you're just starting to program. They tend to be very short, and use a lot of specialized jargon.

    Do not depair! You will learn what a lot of this jargon means throughout this course, and as you get more experienced, just knowing where the error occurs can often be enough for you to spot the problem.

    In the meantime, the most common errors you will see when you are just beginning will be:

    … expected
    not a statement
    reached end of file while parsing
    Something is wrong with your syntax. Check for missing semi-colons and braces.
    cannot find symbol
    You probably misspelled something. This can also happen if you've forgotten to define a variable before using it.
    incompatible type
    You've probably forgotten to use quotes around a string value, put quotes around a number when you shouldn't have, or used single-quotes when you should have used double-quotes. This error will come up for many more situations as you learn to do more with Java.

    Mindprod.com has a nice big list of compilation errors and their likely causes that might also be helpful.

  15. Fix the error in the sample code by removing the line that says "Opps!" (line 4), and save your changes.
  16. Look back at the files in your Project folder. There should now be a build folder next to your src in the main directory. This is where NetBeans stores your compiled code. By default, NetBeans will automatically compile your Java source code into runnable .class files any time you save the file or attempt to run it.

  17. Run the example program by opening it in the NetBeans editor and choosing Run > Run File from the menu (or using the shift-F6 hotkey).

    Why can't I use the big green Run Project button?

    Projects are normally meant to store just one program, and so this button will only run the main method in one of your Java files (called the "main class"). Right now that's fine, but as you add more files to this project it will quickly become less useful.

    If you really like using that button instead of the menu or hotkeys, you can change which class will be run at any time in the settings for your project. Right-click the project name in the Projects panel and choose Properties. Choose Run from the Categories pane and change the Main Class setting as appropriate.

  18. If everything is working normally, you should the following appear in the output panel:

    run:
Success?
BUILD SUCCESSFUL (total time: 0 seconds)

    The black text is the output from this program. The coloured bits are messages from NetBeans letting you know where this output came from, and whether everything was working normally or not.

Task 3: ZIP and Submit

Create a ZIP archive of this lab's package, including the package folder itself.

To create a ZIP file on MacOS:

  1. Open the Finder application
  2. Navigate to the src folder inside your NetBeans project folder
  3. Right-click (or control-click) your lab package folder
  4. Choose Compress "…" from the context menu

To create a ZIP file on Windows:

  1. Open a File Explorer window
  2. Navigate to the src folder inside your NetBeans project folder
  3. Right-click your lab package folder
  4. Choose Send to > Compressed (zipped) folder from the context menu
  5. Type a name for the new ZIP file and press enter to accept

Submit your ZIP file to the Lab 1 assignment in D2L.

All future labs will be submitted in the same way, but will not include a separate task and instructions for this process!

Submission and Grading

Completing the tasks in the lab assignment should result in a ZIP archive of your lab package folder containing a single MyFirstProgram.java file. Check the structure of your ZIP archive carefully – submissions that are not properly structured will not be marked. Submit your ZIP archive to the Lab 1: NetBeans assignment on D2L.
